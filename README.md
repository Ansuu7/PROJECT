# Digital Memory Aid

Full-stack Digital Memory Aid system with:
- Patient and Caretaker dashboards
- Role-based login
- Task tracking, escalation logic, and chat
- Express backend + SQLite database

## Local Run

1. Install dependencies:
   ```bash
   npm install
   ```
2. Start server:
   ```bash
   npm start
   ```
3. Open:
   ```
   http://localhost:3000
   ```

### Demo Accounts
- Patient: `patient1` / `patient123`
- Caretaker: `caretaker1` / `caretaker123`

## Deploy on Render

This repo includes `render.yaml`, so you can deploy directly.

1. Push latest code to GitHub.
2. In Render dashboard, click **New +** → **Blueprint**.
3. Connect your GitHub repo and select this project.
4. Render will read `render.yaml` and create the web service + persistent disk.
5. Wait for deploy to complete.
6. Open the provided Render URL.

### Important Notes
- `SESSION_SECRET` is auto-generated by Render from `render.yaml`.
- SQLite DB and session files are persisted on Render disk at `/var/data`.
- Health check endpoint is `GET /healthz`.

## GitHub Pages + Render (Recommended for your request)

Since this app has a backend, GitHub Pages can host only the frontend files. Keep backend on Render.

1. Deploy backend to Render (from this same repo).
2. After Render deploy, copy your backend URL (example: `https://digital-memory-aid.onrender.com`).
3. Open [frontend-config.js](frontend-config.js) and set:
   ```js
   window.APP_CONFIG = {
       API_BASE: "https://digital-memory-aid.onrender.com"
   };
   ```
4. Push changes to GitHub.
5. Enable GitHub Pages in repo settings:
   - Source: `Deploy from a branch`
   - Branch: `main` / root
6. Wait for GitHub Pages URL (example: `https://ansuu7.github.io/PROJECT/`).

### Render CORS setting

Set Render environment variable:
- `FRONTEND_ORIGIN=https://ansuu7.github.io`

If your GitHub username changes, update `FRONTEND_ORIGIN` accordingly.

## Access from another device

After deployment, open the Render public URL from any device.
